Vula OS Documentation

Everything you need to know to get started with Vula OS.

Installation

Vula OS runs as a Docker container. You need Docker installed on your system.

Quick Start

docker run -p 8080:8080 --shm-size=1g \
  -v vulos-data:/root/.vulos \
  ghcr.io/vul-os/vulos:latest

Then open http://localhost:8080 in your browser.

Note: The --shm-size=1g flag is required for the Chromium browser to work properly. The -v flag persists your data across container restarts.

Docker Compose

services:
  vulos:
    image: ghcr.io/vul-os/vulos:latest
    ports:
      - "8080:8080"
    shm_size: "1g"
    volumes:
      - vulos-data:/root/.vulos

volumes:
  vulos-data:

Platform Notes

  • macOS: Install Docker Desktop. Works on both Intel and Apple Silicon.
  • Linux: Native Docker or Podman. Supports amd64 and arm64.
  • Windows: Use WSL2 with Docker Desktop.

First Boot Setup

On first launch, Vula OS guides you through a setup wizard:

  1. Language — Select your preferred language.
  2. Timezone — Pick your timezone from the interactive map.
  3. Network — Connect to WiFi (optional in Docker).
  4. Account — Create your admin username and password.
  5. Appearance — Choose dark, light, auto, or scheduled theme.
Important: Your login password is also used for sudo in the terminal. Choose a password you'll remember.

Login

After setup, you'll see the login screen. Sign in with your username and password. Your session persists via a cookie, so you'll stay logged in.

Multiple user accounts can be created in Settings > Users & Profiles.

Desktop

The Vula desktop works like a traditional OS with windows, a dock, and a top menu bar.

Window Management

  • Drag the title bar to move windows.
  • Drag corners to resize.
  • Snap to edges: drag to the left/right edge for half-screen, or to the top to maximize.
  • Traffic lights (red/yellow/green) to close, minimize, maximize.

Multi-Desktop

  • Ctrl+1 to 9 — Switch desktops
  • Ctrl+N — Add new desktop
  • The desktop indicator in the menu bar shows which desktop you're on.

Dock

The dock at the bottom shows open windows, the launchpad button, and the chat button. Click the grid icon to open the Launchpad.

Wallpaper

Change your desktop wallpaper in Settings > Appearance > Wallpaper. Upload any image or use the default.

Terminal

Open Terminal from the Launchpad to get a full bash shell.

Session Persistence

Terminal sessions survive window closes and browser refreshes. When you reopen the terminal:

  • If detached sessions exist, you'll see a picker to reattach or create a new session.
  • If no sessions exist, a new one starts automatically.
  • Running processes continue in the background when you close the window.

Sudo

Your terminal user has sudo access. The sudo password is the same as your login password.

sudo apt-get install htop    # Install packages
sudo reboot                  # Reboot the system

Chromium Browser

The built-in browser is a real Chromium instance running inside Vula OS, streamed to your screen via WebRTC.

  • Window controls are in the top-right corner of the browser (no title bar).
  • New tabs/popups open within the same browser window.
  • Drag the thin strip at the top to move the window.

File Manager

Browse and manage files on the system. You can:

  • Navigate directories by clicking folders.
  • Upload files via the upload button.
  • Download, rename, or delete files.
  • Create new directories.

AI Assistant

Vula OS includes a built-in AI chat assistant. Configure it in Settings > AI Assistant.

Supported Providers

  • Ollama — Local AI models. Default: connects to http://host.docker.internal:11434
  • OpenAI — GPT models via API key.
  • Anthropic — Claude models via API key.
  • Custom — Any OpenAI-compatible endpoint.

The AI can generate and run Python code in a sandboxed environment.

Apps & App Hub

Install additional apps from the App Hub. Apps run as isolated services and appear in the Launchpad.

Built-in apps: Terminal, File Manager, Chromium, Activity Monitor, Disk Usage, Drivers, Packages.

Settings

Access Settings from the Launchpad (gear icon). Available sections:

  • AI Assistant — Provider, model, API keys
  • Appearance — Theme, night shift, wallpaper
  • WiFi / Bluetooth / Sound / Display — Hardware settings
  • Battery & Energy — Power modes
  • Users & Profiles — Account management
  • Remote Access — Tunnel for external access
  • About — System info, hardware specs

Fullscreen Mode

For the best experience, use Vula OS in fullscreen mode.

  • Press F11 or click the fullscreen button on the login screen.
  • On macOS: Ctrl+Cmd+F

Firefox: Prevent Esc from Exiting Fullscreen

Firefox users: By default, pressing Esc exits fullscreen immediately. To prevent this:
  1. Type about:config in the Firefox address bar and press Enter.
  2. Click "Accept the Risk and Continue".
  3. Search for: browser.fullscreen.exit_on_escape
  4. Double-click the preference to set it to false.

With this setting, use F11 or the browser menu to exit fullscreen instead of Esc.

Data Persistence

Mount /root/.vulos to persist user data, settings, and databases across container restarts:

docker run -v vulos-data:/root/.vulos ...

Multi-Architecture

Docker images are built for both linux/amd64 and linux/arm64 architectures.

Troubleshooting

Browser shows black screen

Make sure you're running Docker with --shm-size=1g. Chromium needs shared memory to function properly.

Terminal shows container hostname

Log out and log back in. On login, the system syncs your Linux user account and creates a proper shell profile.

Cannot connect to AI

If using Ollama locally, ensure it's running and accessible at http://host.docker.internal:11434 from inside Docker. On Linux, you may need --add-host=host.docker.internal:host-gateway.

Forgot password

You can reset by deleting the auth database: docker exec vulos rm /root/.vulos/db/auth.json then restart the container.
